## 1. Product Inventory System (Intersection + Template Literal Types)

A system stores products with dynamic SKU values.

Each product has:

* name
* price
* sku
* category

SKU must follow this format:

* SKU-${number}

Categories can only be:

* electronics
* clothing
* food

Task:

* Create a Category union type
* Create a SKU template literal type
* Create a Product interface
* Create a type that combines Product with:

  * stock (number)
  * isAvailable (boolean)

## 2. School System (Intersection of Multiple Roles)

A school stores users who can have multiple roles.

Common properties:

* id
* name
* age

Student-specific:

* level
* courses (array of strings)

Teacher-specific:

* subject
* yearsOfExperience

Task:

* Create a Common type
* Create StudentRole and TeacherRole
* Create:

  * Student (Common & StudentRole)
  * Teacher (Common & TeacherRole)
* Create an array that stores multiple teachers

## 3. Payment System (Union + Narrowing)

A payment system supports multiple payment methods.

Payment methods:

* card
* bank
* crypto

Each payment contains:

* amount
* currency
* method

If method is:

* card → include cardNumber
* bank → include bankAccount
* crypto → include walletAddress

Task:

* Create a PaymentMethod union
* Create separate types for each method
* Combine them into a single Payment union type
